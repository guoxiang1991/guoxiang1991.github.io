HTML5

  • 通俗的讲，HTML5就是一个对于HTML的版本号而已，没有什么神奇的。但是，他又不仅仅只是一个版本号，他更重要的作用是指定了一系列的Web开发的标准，成为第一个将Web做为应用开发平台的HTML语言。

    H5的新特性

    1. 语义化更强–新的特殊内容元素，比如 article、footer、header、nav、section。
    2. 新的表单控件，比如 calendar、date、time、email、url、search。
    3. 用于媒介回放的 video 和 audio 元素。
    4. 对本地离线存储的更好的支持。
    5. 用于绘画的 canvas 元素。

HTML5的语义化

  • 既然HTML5中有很多新特性，下面我们就来一点点的认识一下。

HTML5中的新元素（语义化）

  • HTML5中添加了很多新的元素，也就是标签。比如 article、footer、header、nav、section。这些也就仅仅只是一些标签而已，和我们以前接触的 div、p、span这些一样，仅仅是一个标签，没有任何的样式附加，只是为了增强语义化，为了便于搜索引擎和辅助设备更好的理解我们所编写的HTML。

    • 下面我们来看一下都有哪些新的元素：

      • header（页眉）

      • footer（页脚）

      • nav（导航）

      • section（区块）

      • article（文章）

      • aside（侧边栏）

      • progress（进度条）

      • ……

    • 从本质上来说，他们于以前的div、span没有任何区别，只是语义性更强。

    • 对于我们传统的开发者，我们在布局的时候经常这样写

    • 这样写没有错误，也没有什么不好，但是我们没有硬性的规定每一个标签的class名，因为class名是每一个开发者根据子级的心意来写，所以大家就有可能不一样。

    • 而HTML5中就给我们做了一个规定，把我们以前的class名变成了一个标签名来使用，这样就避免了大家书写时候的不一样，而搜索引擎和辅助设备在抓取的时候的可以直接根据标签名来分辨。

angular的自定义指令

自定义指令

  • 在 angular 中有很多的内置指令供我们使用；通过使用这些指令我们可以更加方便快捷的操作各种数据。但是仅仅这些内置的指令已经不能满足我们在开发中的需求了，我们想要更快更好的完成一些项目，我们就应该根据我们平时开发中的需求自己来添加一些自定义指令，这样可以是我们的开发效率更高。其实自定义指令就像以前我们为了实现某些功能而封装一些方法一样。

创建指令

  • 创建一个指令我们使用 myApp.directive() 这个方法

    • 和以往的操作一样，我们先要创建一个模块，一切从模块开始吗！
    • 然后我们使用 myApp.directive() 这个方法直接创建指令就好
    • 在我们自定义的指令中 myInstructions 是我们自定义的这个指令的名称，就好像内置指令中的 ng-model一样。但是我们要注意一点，我们在定义这个自定义指令的时候，指令名称要用驼峰式命名法来命名，而我们在使用我们定义好的这个指令的时候要把驼峰式命名法中的每个单词拆开，用‘-’连接，并且单词字母全部是小写。例如：我们定义一个指令的时候写为 ‘myInstructions’ ，而使用一个指令的时候要写成 ‘my-instructions’ 才可以，不然是不会成功的。
    • 定义指令以后我们需要的是后面函数中的返回值，返回值要是一个对象。

定义指令

  • 我们定义一个指令主要是在 function 中的 return 的对象上进行定义

    • template：定义一个模版(指的是HTML结构)，定义好的结构模版会根据我们的操作显示在页面中
    • replace：默认值为false，意思是会在页面中使用当前自定义指令的标签下新建 template 中定义的HTML结构，如果写 true 的意思是 template 中定义的HTML结构会直接覆盖掉使用当前自定义指令的标签。
    • restrict：是指我们自定一的标签使用的位置，默认为 ‘EA’。在这里不多解释，后面的 ‘使用指令’ 中会详细解释
      E element 用于标签
      C class 用于样式类名
      M comment 用于注释
      A attribute 用于属性名
    • transclude：默认为false，意思是当我们把原有标签覆盖掉以后不会保存原有标签中的内容。一般和内置指令 ng-transclude 结合使用，如果写 true 的意思就是把原有标签中的内容添加进来。
    • scope：默认为false，不会形成私有的作用域。如果为true的话就会形成一个私有的作用域
    • link：实现和DOM元素、作用域的连接(连接阶段)。这个也是最重要的一部分，我们自定义指令的目的就是在这个方法里实现一些任务以后，直接返回给DOM元素。例如：我们在这个 function 中放入一个 ‘拖拽’ 的方法，那么我们使用这个自定义指令的标签就会实现拖拽的功能，在这个 function 中的代码基本已经可以说是回归了 ‘原生JS’ 代码了。我们只需要用 ‘原生JS’ 的方法写就可以实现功能。当然，我们也可以使用 jQuery 来写，如果我们要使用jQuery写的话我们最好是通过 ‘script’ 标签把jQuery引入进来，因为我们的angular框架中只有轻量级的jQuery代码。在这里使用的时候要注意，我们这个函数内的三个形参，我们函数里写的方法要根据这三个形参来写。
      scope 当前模版的作用域（如果当前没有自己形成的作用域的话会自动找父级）
      element 当前指令所在的DOM元素，是一个jQuery对象
      attrs 自定义指令所在元素上的属性存储在attrs对象中

  • 就像这样，我们可以直接在link这个function中写如一些方法，在页面中使用当前自定义指令以后，使用该自定义指令的标签就可以进行拖拽了。如果该标签没有写过CSS样式的话，不要忘了写CSS样式。

使用指令

  • 我们使用一个指令主要是在HTML页面中直接使用就好，自定义指令也是一样的，直接使用就行。

    • 我们在之前的定义阶段中说过自定义指令的使用位置有四个 ‘ECMA’ 在这里我们要解释一下怎么使用

      1. E element 用于标签

      2. C class 用于样式类名

      3. M comment 用于注释

      4. A attribute 用于属性名

    • 以上四种方法都可以，但是第三种用于注释我们一般不这么使用。
