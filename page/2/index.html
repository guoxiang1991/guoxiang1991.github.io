H5API-drag

H5的新增API中的drag

  • 这里所说的拖拽和以前我们结束到的差不多，但是还是不太一样，这个拖拽主要是用来和刚才的文件读取配合使用，因为我们不光可以点击选择图片上传，也可以使用拖拽的方式把图片直接拖拽到页面上进行上传。
  • 首先来解释一下什么是拖拽元素和目标元素：
    • 拖拽元素：就是指我们想拖动哪个元素，哪个元素就是拖拽元素（其实就是我们在页面上是拖着谁走谁就是）。
    • 目标元素：我们要把我们的“拖拽元素”拖到哪，那就是目标元素。
  • 这个时候我们就需要一些事件来把拖拽这个事情完成
    • 应用于“拖拽元素”的几个事件：
    • ondrag：拖拽时调用（就是我们拖着一个“拖拽元素”在页面上来回跑的时候，每当位置发生一次改变，就会触发）。
    • ondragstart：拖拽开始时调用（注意这个开始并不是鼠标点击下去就算开始，而是按下去以后并且发生了位移）。
    • ondragend：拖拽结束时调用（就是我们已经改变了“拖拽元素”的位置以后松开鼠标才会触发）。
    • ondragleave：鼠标离开拖拽元素时调用（我们的“拖拽元素不管多大都是有个范围的，当光标离开范围的时候触发”）
  • 刚才说的就是“拖拽元素”的事件，这里有一个需要注意的点就是当拖拽开始的时候，虽然我们看到的是一个完整的“拖拽元素”，但是这个时候我们的拖拽元素实际上只有“光标的尖点”那么大。也就是说只有“光标的尖点”才是我们真是的拖拽元素，而不是我们看到的整个元素。而当我们松开鼠标以后，我们的“拖拽元素”就不再是“光标的尖点”了，而是我们看到的那个整个元素，知道他回到一开始的位置才结束。
  • 刚才说了拖拽元素的一些事件，接下来我们说一下目标元素的事件，因为任意一个元素都可以是目标元素，所以我们想让谁当目标元素的时候就用谁来绑定这些事件
    • 应用于“目标元素”的几个事件：
    • ondragenter：当“拖拽元素”进入“目标元素”后触发（注意是“光标的尖点”进入目标元素时触发）。
    • ondragover：当“拖拽元素”在“目标元素”内就会触发（注意是“光标的尖点”在目标元素中的时候触发，而且是不管有没有位移都会触发）
    • ondragleave：当“拖拽元素”离开“目标元素”以后触发（注意是“光标的尖点”离开目标元素的时候触发）。
    • ondrop：当“拖拽元素”在“目标元素”内松开鼠标时触发（注意是“光标的尖点”在目标元素内松开时触发）。
  • 在这里有一个需要注意的地方就是当我们在“目标元素”内松开鼠标的一瞬间，我们的“拖拽元素”会变成我们看到的整个元素，而在这个整个元素回到开始位置的过程中还是会触发“ondragover”事件，所以我们如果绑定了“ondragover”事件就一定要加上一个“preventDefault()”方法来阻止“ondragover”的默认事件。
  • 当我们从本地拖拽一个图片到我们页面上的目标元素上松开鼠标时，会触发一个ondrop事件的默认事件，就是以一个新标签页打开图片地址，这个时候我们也需要给我们的ondrop事件来加上一个“preventDefault()”方法阻止他的默认事件。
  • 这样我们的拖拽元素和目标元素的事件就都有了，我们就可以根据这些事件来进行拖拽文件读取了。

  • 这个过程就和之前的文件读取一样了，只不过这个时候存储我们图片信息的位置不一样了。

  • 这个“dataTransfer”里面就存储了“files”，和刚才的道理一样，只不过刚才我们直接是“file”对象下有一个“files”，而在拖拽上传中，我们的“files”藏在了“dataTransfer”里面，所以我们就需要多拐一个弯而已

  • 这里我们得到的res就是我们拖拽的图片的具体信息了，然后我们得到了具体信息以后就可以像刚才读取文件时候的步骤一样来绑定了。

  • 现在我们就完成了拖拽图片上传的事情，其实和之前的文件读取一样，只不过是通过一些drag事件来完成的。

H5API-geolocation

H5的新增API中的geolocation

  • 这个地理定位在移动端经常用于导航，PC端一样，也是用于地图上，比如我们的百度地图中。
  • 因为这个API我们没有办法自己制作，所以说我们只要去百度开放的接口直接使用就好了。
  • 不管我们用在移动端还是PC端，我们要做的第一件事都是进行定位，只有知道当前使用设备的准确位置，才可以进行下一步操作。

  • 这里H5给我们提供了两个方法，用来让我们获取用户的位置信息。

  • 这两个方法就是我们获取用户位置信息的方法，当然一定是要在用户点击同意以后才可以获取，不然的话是获取不到的。

  • 这两个方法都是全局对象window下的navigator下的geolocation下的方法，我们调用的时候要写全，但是navigator这个对象因为实在全局对象下的，所以window可以省略，但是建议写上window，只是一种严谨的书写模式，当然不写也没事。

    • getCurrentPosition() 是获取当前用户的位置信息。（只获取一次信息用在我们进行定位的时候）
    • watchPosition() 是重复获取当前用户信息。（随着你设置的重复获取间隔事件来获取位置信息，比如导航的时候）
    • 两个方法后面分别有三个参数
      • successCallback 是获取位置信息成功以后的回调函数
      • errorCallback 是获取位置信息失败以后的回调函数
      • options 是配置参数（这个参数在我们不需要使用的时候是可以不写的，他是一个对象，叫做对象传参，一般用作配置使用，这里知道怎么使用就行，后期讲ajax的时候封装一个ajax方法的时候就会学到了。）
    • 这里我们就知道一下他的使用方法就好了，没必要纠结
      • 他里面可以配置三个参数值
      1. enableHighAccuracy 高精度模式（就是指我们获取位置的时候要不要启用高精度模式）
      2. timeout 设置一个超时时间（就是拟规定一个时间，在这个时间内不能获取位置信息就中断，注意单位是“ms”）
      3. maximumAge 浏览器重新获取位置的时间间隔（一般用在重复获取的时候设置，注意单位也是“ms”）
    • 这两个方法不管使用哪个，在当前浏览器第一次使用的时候，他都会自动询问用户是否允许，浏览器会记下你是否允许然后进行后续的操作。

  • 这就是具体的使用方法就是这样，当然“watchPosition()”方法在使用的时候和“getCurrentPosition()”是一样的。

  • 在成功回调里出现的“options”和失败回调里出现的“err”都是函数执行是自动传给我们的一个参数，就和我们的事件对象“event”是一样的道理。
    • options 里面存储的是我们所获得的位置信息
    • err 里面存储的是我们失败的原因
  • 这里一般我们不会成功，因为在我们国内是受到了限制，是不允许我们得到位置信息的，所以实验时我们一定会进入错误回调。
  • 开发中我们是不会遇到这个问题的，因为人家有授权，而我们没有。

  • 假设现在我们成功了，我们就能得到一个“options”，而options下面会有一个coords的对象，这个对象上就存在了几个我们需要用的到属性

    • latitude 纬度
    • longitude 经度
    • accuracy 精确度（就是指我们得到的位置信息大概在一个什么范围内）
    • altitude 海拔高度

  • 这段代码放在成功的回调里执行，我们就会得到对应的我们需要的位置信息

  • 因为我们没法自己制作一张准确的地图，所以我们就使用百度的接口进行下面的操作就好了，我们自己需要做到的就是获取一下当前的位置信息。
  • 下面我们来解释一下百度接口的使用。
    1. 首先我们打开百度地图第三方API
    2. 点击导航栏里的“javaScript API”就行
    3. 点击左侧侧边栏的“开发指南”，然后选择“Hello World”就好
    4. 里面会有实例告诉我们具体怎么操作，按照他的实例我们自己写就行了。

  • 再来写一下我们具体怎么写

    1. 引入百度规定好的JS文件“http://api.map.baidu.com/api?v=2.0&ak=你的密钥”

    2. 创建一个获取位置信息的函数，书写是尽量把回到函数写在外面，代码的可读性更高，培养一个良好的书写习惯

    3. 咱们就直接在成功的回调函数“success”下继续咱们的操作就可以了

  • 这样我们就能获取到位置信息后在地图上反映出来

H5API-history

H5的新增API中的history

  • 是在我们的历史区添加一些记录，使我们的页面可以在历史区暂存，也可以让已经存在历史区的页面被替换掉，这样我们点击回退的时候就不会回到我们刚才来的那个网页，而是会回到我们替换好的另一个网页。
  • 其实历史区和我们的数组很像，所以我们记忆的时候只要类比着数组记忆就可以。（目前我们只要记住两个方法一个事件就可以）

  • 第一个是pushState()方法。是向我们的历史区中追加一条记录。

  • 一般来说这个方法都是绑定在某个事件中，基本不会直接调用。其实和数组中的push方法一样。

  • 第二个方法是replaceState()方法，是替换掉当前的历史记录，而不是在后面追加了，使用方法和之前的pushState()一样。

  • 这个也是绑定在某个事件中，基本不会直接调用。

  • 这就是我们的两个方法，很好用，也很方便实用，但是要记得方法调用一定是通过history调用。

  • 接下来我们就说一下事件，onpopstate事件。是在历史区发生改变后，并且我们操作历史区的时候才会触发。

  • 这个事件是window下的事件，所以一定要写成window.onpopstate。

  • 这里一定要注意，我们触发这个事件一定要满足两个条件，一是我们的历史区发生了改变，二是改变后我们操作历史区的时候。

H5API-storage

H5的新增API中的storage

  • storage是我们的WEB存储，分为localStorage和sessionStorage两种，和我们以前的cookie相比，这两种存储机制更强大，设置和读取都比以前方便很多，存储容量也变的更大。
  • localStorage： 大约可存储5M的内容，浏览器窗口关闭则这里存储的内容消失，只能存储字符串。
  • sessionStorage： 大约可存储20M的内容，是永久存储，不会随着浏览器的关闭而消失，除非手动删除，只能存储字符串。
  • 因为只能存储字符串，所以我们要存储对象的时候要将对象通过JSON.stringify()编码后存储。

  • 他们的方法都是一样的，也就是说我们用谁调用就存在谁的名下了

  • sessionStorage和localStorage的方法是一样的，只不过是用sessionStorage来调用。

  • 其实我们就把他当成一个普通对象看待就好了，就像我们操作一个普通对象一样简单。

  • 我们只要注意sessionStorage的生命周期是浏览器窗口关闭后消失，而localStorage则是永久有效，除非手动删除就好了。

angular的自定义指令

自定义指令

  • 在 angular 中有很多的内置指令供我们使用；通过使用这些指令我们可以更加方便快捷的操作各种数据。但是仅仅这些内置的指令已经不能满足我们在开发中的需求了，我们想要更快更好的完成一些项目，我们就应该根据我们平时开发中的需求自己来添加一些自定义指令，这样可以是我们的开发效率更高。其实自定义指令就像以前我们为了实现某些功能而封装一些方法一样。

创建指令

  • 创建一个指令我们使用 myApp.directive() 这个方法

    • 和以往的操作一样，我们先要创建一个模块，一切从模块开始吗！
    • 然后我们使用 myApp.directive() 这个方法直接创建指令就好
    • 在我们自定义的指令中 myInstructions 是我们自定义的这个指令的名称，就好像内置指令中的 ng-model一样。但是我们要注意一点，我们在定义这个自定义指令的时候，指令名称要用驼峰式命名法来命名，而我们在使用我们定义好的这个指令的时候要把驼峰式命名法中的每个单词拆开，用‘-’连接，并且单词字母全部是小写。例如：我们定义一个指令的时候写为 ‘myInstructions’ ，而使用一个指令的时候要写成 ‘my-instructions’ 才可以，不然是不会成功的。
    • 定义指令以后我们需要的是后面函数中的返回值，返回值要是一个对象。

定义指令

  • 我们定义一个指令主要是在 function 中的 return 的对象上进行定义

    • template：定义一个模版(指的是HTML结构)，定义好的结构模版会根据我们的操作显示在页面中
    • replace：默认值为false，意思是会在页面中使用当前自定义指令的标签下新建 template 中定义的HTML结构，如果写 true 的意思是 template 中定义的HTML结构会直接覆盖掉使用当前自定义指令的标签。
    • restrict：是指我们自定一的标签使用的位置，默认为 ‘EA’。在这里不多解释，后面的 ‘使用指令’ 中会详细解释
      E element 用于标签
      C class 用于样式类名
      M comment 用于注释
      A attribute 用于属性名
    • transclude：默认为false，意思是当我们把原有标签覆盖掉以后不会保存原有标签中的内容。一般和内置指令 ng-transclude 结合使用，如果写 true 的意思就是把原有标签中的内容添加进来。
    • scope：默认为false，不会形成私有的作用域。如果为true的话就会形成一个私有的作用域
    • link：实现和DOM元素、作用域的连接(连接阶段)。这个也是最重要的一部分，我们自定义指令的目的就是在这个方法里实现一些任务以后，直接返回给DOM元素。例如：我们在这个 function 中放入一个 ‘拖拽’ 的方法，那么我们使用这个自定义指令的标签就会实现拖拽的功能，在这个 function 中的代码基本已经可以说是回归了 ‘原生JS’ 代码了。我们只需要用 ‘原生JS’ 的方法写就可以实现功能。当然，我们也可以使用 jQuery 来写，如果我们要使用jQuery写的话我们最好是通过 ‘script’ 标签把jQuery引入进来，因为我们的angular框架中只有轻量级的jQuery代码。在这里使用的时候要注意，我们这个函数内的三个形参，我们函数里写的方法要根据这三个形参来写。
      scope 当前模版的作用域（如果当前没有自己形成的作用域的话会自动找父级）
      element 当前指令所在的DOM元素，是一个jQuery对象
      attrs 自定义指令所在元素上的属性存储在attrs对象中

  • 就像这样，我们可以直接在link这个function中写如一些方法，在页面中使用当前自定义指令以后，使用该自定义指令的标签就可以进行拖拽了。如果该标签没有写过CSS样式的话，不要忘了写CSS样式。

使用指令

  • 我们使用一个指令主要是在HTML页面中直接使用就好，自定义指令也是一样的，直接使用就行。

    • 我们在之前的定义阶段中说过自定义指令的使用位置有四个 ‘ECMA’ 在这里我们要解释一下怎么使用

      1. E element 用于标签

      2. C class 用于样式类名

      3. M comment 用于注释

      4. A attribute 用于属性名

    • 以上四种方法都可以，但是第三种用于注释我们一般不这么使用。
